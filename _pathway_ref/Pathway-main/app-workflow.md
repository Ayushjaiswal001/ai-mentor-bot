# PathwayAI — App Workflow

A one-page tour. Short on purpose.

## What it is

A Telegram bot that coaches one engineer toward being interview-ready. It
runs in three loops: **daily**, **per-message**, and **weekly**.

---

## Daily loop

| Time | What happens |
|---|---|
| **Morning** | Bot posts a check-in. Pulls yesterday's logs, due topics, user instructions ("won't study today"), and current streak. Asks for `/goals`. |
| **Day** | You chat freely or use commands. Every `/log` becomes a structured row (topic, difficulty, tradeoff, interview story). |
| **Evening** | Bot scores today's goal as `completed / partial / skipped`, sends 🎯 + streak when completed, and posts a reflection question. |
| **20:00 (cron)** | Daily nudge fires for anyone who didn't log today. |

---

## Per-message flow

```
Telegram update
   │
   ▼
dedup by update_id  ──► duplicate? drop.
   │
   ▼
allowlist check  ──► not allowed? drop.
   │
   ▼
save inbound message
   │
   ▼
rate limit check (60/hr default)
   │
   ▼
interaction state?  ──► yes ──► quiz / mock / onboarding / log-edit handler
   │ no
   ▼
slash command?  ──► yes ──► command branch
   │ no
   ▼
free text  ──► _chat (uses last 12 messages + memory + logs + goals)
              optionally compacts old messages into chat_summary
```

Every reply runs through `<USER_INPUT>` delimiters; user text is treated
as data, never as instructions.

## Commands

### Core
- `/goals <text>` — set today's goal. `/goals` alone shows current + recent
  with **Clear** button.
- `/goals dsa | systems | review | project` — templated goal.
- `/log <what you built or learned>` — saves it instantly, extracts topic
  + difficulty + STAR-style story in the background, then sends a
  `🔥 Streak: N days` follow-up.
- `/logs` — last 5 with **Edit / Delete** buttons per row.
- `/ask <question>` — one-shot technical Q&A.

### Skill tracking
- `/mastery` — per-topic level, last-quizzed date, due dates. Topics that
  are due now get a `Quiz: topic` button.
- `/review` — auto-quizzes the most-overdue topic.
- `/mock <topic>` — five-turn mock interview. Final rubric saved as memory
  and updates topic mastery. `/end` to stop early.
- `/mock` alone — suggests your 3 weakest or due topics.

### Reflection / output
- `/stories` — interview stories grouped by topic, with `Quiz Me` per row.
- `/codereview <code>` — structured review (`STRENGTHS / BUGS /
  COMPLEXITY / INTERVIEW_NOTES`), saved as a `code_review` log.
- `/export week | month` — markdown bundle as a Telegram document.
- `/status` — log streak, 7-day heatmap, goal stats, readiness subscores +
  gaps.
- `/next` — current weekly plan.
- `/activity` — last 7 days of GitHub / LeetCode events grouped by source.

### Lifecycle
- `/start` — first run launches 3-question onboarding; pins the welcome.
- `/help` — compact list. `/help all` — full list.
- `/version` — running build SHA.
- `/forgetme confirm` — wipe all data.

### Free text
Anything that isn't a slash command goes into `_chat` — a real
conversation that remembers your prior messages, today's goal, recent
logs, and prior memory entries (including `user_instruction`s like "won't
study this week").

## Data model (only what matters)

- **users** — one per Telegram chat
- **conversation_messages** — chat history; oldest get rolled into
  `chat_summary` after 40 messages
- **learning_logs** — structured: `built / topic / difficulty / tradeoff /
  interview_story / extraction_status`
- **daily_goals** — `status: planned | completed | partial | skipped`
- **topic_mastery** — `level + next_due_at`, spaced repetition keyed on
  log topics
- **memory_summaries** — typed: `quiz_assessment`, `goal_outcome`,
  `mock_interview`, `user_instruction`, `chat_summary`, `profile`
- **readiness_scores** — weekly snapshot with subscores + gap analysis
- **weekly_plans** — generated by the weekly workflow
- **activity_events** — synced GitHub / LeetCode
- **processed_updates** — webhook dedup
- **model_call_logs** — every LLM call: provider, success, tokens,
  latency

---

## LLM layer

- **Gateway** with provider order Groq → HuggingFace → deterministic
  fallback string.
- JSON mode used for extraction and goal-outcome.
- Every call logged with latency + token counts → surfaces in
  `/admin/model-usage`.
- Failed calls retry with exponential backoff.

---

## Workflows (scheduled, LangGraph)

- **Morning** — load context (events, logs, goal, instructions, due
  topics, streak) → compose → deliver.
- **Evening** — load context → score today's goal → compose reflection →
  deliver. Goal completion fires a 🎯 + streak message.
- **Weekly** — evaluate (events + logs + memories → deterministic
  readiness score) → compose review → persist plan + memory → deliver.

---

## Admin / health

- `GET /health` — status, version, last model call timestamp + success.
- `GET /admin/model-usage` — JSON or HTML. HTML view shows per-provider
  success rate, token totals, latency, 7-day sparkline, recent errors.
- `POST /admin/daily-nudge` — wire to a 20:00 cron.
- `POST /admin/prune?days=90` — wire to a weekly cron.
- Every `/admin/*` call is audit-logged with caller IP.

---

## Safety

- **Dedup** — every `update_id` and `callback_query.id` is single-use.
- **Allowlist** — `TELEGRAM_ALLOWED_CHAT_IDS` gates who can talk to the
  bot.
- **Rate limit** — `chat_rate_limit_per_hour` (60 by default).
- **Prompt-injection guard** — user text wrapped in `<USER_INPUT>` tags;
  system prompt instructs the model to treat them as data.
- **PII redaction** — structlog processor scrubs emails, phones, tokens,
  high-entropy blobs before any log line is emitted.
- **`/forgetme confirm`** — wipes every user-scoped row in one call.
